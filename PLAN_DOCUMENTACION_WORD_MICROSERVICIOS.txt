PLAN PARA DOCUMENTAR LOS MICROSERVICIOS EN WORD

Objetivo:
Crear una documentacion tecnica defendible del proyecto juego-tienda, usando
Swagger/OpenAPI, codigo fuente, pruebas y evidencia de despliegue.

Estrategia recomendada:
Hacer un Word maestro llamado:
Documentacion_Tecnica_Microservicios_Tienda_Videojuegos.docx

Ese Word debe contener todos los servicios en un solo documento. Es mejor que
tener muchos documentos separados, porque en la defensa permite buscar rapido
y mostrar arquitectura, endpoints, Docker, Swagger y pruebas desde un solo lugar.

3. ENDPOINTS REALES POR MICROSERVICIO

VIDEOJUEGOS
Base path: /videojuegos
- GET /videojuegos
- GET /videojuegos/{id}
- POST /videojuegos
- PUT /videojuegos/{id}
- DELETE /videojuegos/{id}
- GET /videojuegos/buscar

Puntos a explicar:
- Catalogo de videojuegos.
- Busqueda por plataforma.
- Busqueda por rango de precio.
- Validacion de plataforma.
- Error personalizado para plataforma invalida.


USUARIOS
Base path: /usuarios
- GET /usuarios
- GET /usuarios/{id}
- GET /usuarios/buscar
- POST /usuarios
- PUT /usuarios/{id}
- DELETE /usuarios/{id}

Puntos a explicar:
- Gestion de cuentas de usuario.
- Se muestran nombres y datos utiles, no solo IDs.
- Es usado por authentication, carrito, pagos, pedidos y resenas.


AUTHENTICATION
Base path: /auth
- POST /auth/registro
- POST /auth/login
- GET /auth/credenciales
- GET /auth/credenciales/{id}
- PUT /auth/credenciales/{id}/password
- DELETE /auth/credenciales/{id}

Puntos a explicar:
- Registro de cuenta.
- Login simple sin JWT/Spring Security.
- Password obligatorio.
- Password minimo 5 caracteres.
- Error personalizado para contrasena invalida.
- No se expone passwordHash en la lista de credenciales.

5. COMO GENERAR LA DOCUMENTACION DESDE SWAGGER

1. Levantar el proyecto:
   docker compose up --build -d

2. Abrir Swagger:
   http://localhost:8080/swagger-ui/index.html

3. Para cada API:
   - Seleccionar el microservicio en el selector superior.
   - Copiar endpoints principales.
   - Probar un GET y un POST/PUT si corresponde.
   - Guardar captura de request/response.

4. Descargar o consultar OpenAPI JSON:
   - /videojuegos/v3/api-docs
   - /usuarios/v3/api-docs
   - /auth/v3/api-docs
   - /carrito/v3/api-docs
   - /pagos/v3/api-docs
   - /pedidos/v3/api-docs
   - /resenas/v3/api-docs
   - /inventario/v3/api-docs

5. Apagar:
   docker compose down --remove-orphans

7. CRITERIO PARA LA DEFENSA

Para cada microservicio debes poder responder:
- Que problema resuelve.
- Que endpoint principal tiene.
- Que datos recibe y que devuelve.
- Que validacion importante aplica.
- Que error personalizado puede devolver.
- Que otro microservicio consume o quien lo consume a el.
- Como se ve en Swagger.
- Que prueba confirma que funciona.

La frase clave para defender:
"El API Gateway centraliza la entrada, Eureka permite descubrir servicios,
Config Server centraliza configuracion YAML, cada microservicio mantiene su
responsabilidad de negocio, y Swagger/Postman permiten probar los endpoints de
forma documentada."
