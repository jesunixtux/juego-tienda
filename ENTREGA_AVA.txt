Repositorio GitHub: https://github.com/jesunixtux/juego-tienda

Contenido incluido:
- Microservicios Spring Boot.
- API Gateway.
- Eureka Server.
- Config Server.
- Archivos application.yml.
- Dockerfile y docker-compose.yml.
- Apache con PHP y phpMyAdmin.
- Coleccion Postman.
- README.md.
- GUIA_ESTUDIO.md.
- RESULTADOS_PRUEBAS_DEFENSA.txt.
- Scripts para ejecutar local sin Docker y para Docker.
- Pruebas JUnit/Mockito e integracion.
- Swagger/OpenAPI actualizado sin seguridad obligatoria.
- Swagger/OpenAPI con Try-it-out habilitado y CORS local para Execute.
- API Gateway sin Spring Security; enruta los microservicios por Eureka.
- Creacion de cuentas con password obligatoria, minimo 5 caracteres y error personalizado.

URLs locales principales:
- API Gateway: http://localhost:8080
- Swagger: http://localhost:8080/swagger-ui/index.html
- Eureka: http://localhost:8761
- Config Server ejemplo: http://localhost:8888/videojuegos/default
- Apache/PHP: http://localhost:8081
- phpMyAdmin: http://localhost:8082

Comandos principales:

1. Levantar local sin Docker en macOS/Linux:
   bash scripts/local-up.sh

2. Detener local sin Docker en macOS/Linux:
   bash scripts/local-down.sh

3. Levantar local sin Docker en Windows PowerShell:
   powershell -ExecutionPolicy Bypass -File scripts/local-up.ps1

4. Detener local sin Docker en Windows PowerShell:
   powershell -ExecutionPolicy Bypass -File scripts/local-down.ps1

5. Levantar con Docker:
   docker compose up -d --build

6. Ver contenedores:
   docker compose ps

7. Ver logs del Gateway:
   docker compose logs -f api-gateway

8. Probar stack:
   BASE_URL=http://localhost:8080 EUREKA_URL=http://localhost:8761 CONFIG_URL=http://localhost:8888 bash scripts/test-definitivo.sh

9. Detener:
   docker compose down

Notas:
- El puerto MySQL local de Docker es 3307 para no chocar con XAMPP/MySQL local en 3306.
- El modo local sin Docker espera MySQL en localhost:3306.
- Dentro de Docker los microservicios se conectan a mysql:3306.
- La password MySQL por defecto esta vacia, salvo que se cambie en .env.
- No se usa Spring Security ni tokens obligatorios; el Gateway queda abierto para la evaluacion.
- La carpeta backup-tienda no se incluye en el ZIP de entrega.
- Las carpetas target, .git y docker/maven-repository no se incluyen en el ZIP para reducir peso.
